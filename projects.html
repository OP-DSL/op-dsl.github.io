Potential future projects

  • Visualisaiton
    • Type: new core feature
    • Difficulty: medium
    • Description: visually analysing the results of the simulation is key for scientific insight. This can be done either off-line, once the simulation finished, or on-line as the simulation is progressing. This project involves understanding the VTK format and data structures, and implementing support for writing datasets to file for off-line visualisation, followed by doing this on-line using the Catalyst framework
    • Mentor: Istvan Reguly
  • Continuous integration
    • Type: build system
    • Difficulty: medium
    • Description: continuous integration is key in long-term code development and the maintenance of our codes. The project involves setting up unit and regression tests for different parts of the code, and integration into the github environment, to automate these tasks.
    • Mentor: Gihan Mudalige
  • CMAKE build system
    • Type: build system
    • Difficulty: easy
    • Description: OPS and OP2 currently mainly use GNU make to build the source code, and support different compilers and platforms. The project would involve developing cmake support, that can detect and work with different discributed memory environments, compilers (GNU, Clang, PGI, Intel, IBM XL), customise flags for a variety of compiler-parallelisation combinations, and hardware (instruction sets in CPUs, different GPU generations).
    • Mentor: Istvan Reguly
  • Tridiagonal solver
    • Type: core feature
    • Difficulty: medium
    • Description: tridiagonal equations commonly come up in structured meshes, when using implicit solvers. A common algorithm is ADI (Alternating Direction Implicit) which solves these equations along different axes when dealing with a 2 or larger dimensional problem. The Thomas algorithm and the Parallel Cyclic Reduction are some of the commonly used algorithms to solve this. The goal of the project is to extend these algorithms to work in a distributed memory environment, to provide a globally coherent solution.
    • Mentor: Gihan Mudalige
  • Just-in-Time compilation use cases
    • Type: extension
    • Difficulty: hard
    • Description: JIT is a core feature being developed into OPS and OP2, which allows to re-compile certain (performance-sensitive) parts of the code at run-time, at which point several parameters are known and fixed. The project will explore different optimisations that can be applied this way: hard-coding loop bounds, runtime constants, applying loop fusion or splitting, using external tools such as polyhedral compilers, etc.
    • Mentor: Gihan Mudalige
  • Hybrid CPU/GPU execution
    • Type: core feature
    • Difficulty: medium
    • Description: OPS and OP2 can already use CPUs or GPUs, but not yet together. A key challenge is that the relative performance between the two changes between computational phases, therefore load balancing should be highly dynamic. This project will explore dynamic load balancing techniques and libraries that can be integrated into OPS or OP2.
    • Mentor: Istvan Reguly
  • Conversion tool for Fortran
    • Type: compilers
    • Difficulty: hard
    • Description: converting legacy applications to use the OPS and OP2 APIs is relatively strightforward for most codes, but is still a very manual process. This project will look into using Flang to analyse simple tightly nested loops and determine access patterns, then outline the loop body and replace it with a call to OPS or OP2.
    • Mentor: Gihan Mudalige
